<objective>
Implement admin-configurable field groupings that dynamically expose grouped job arrays as Etch template variables.

Purpose: Replace the hardcoded cws_jobs_by_category and cws_jobs_by_city groupings with a flexible admin-controlled system. Site editors can define any job field (e.g. "primary_category", "primary_city", "department") as a grouping source and the plugin automatically exposes {options.cws_jobs_by_{field}} in Etch templates, keyed by unique field values with arrays of matching jobs as values.

Output:
- Repeater UI in the API Configuration settings section for entering field names (add/remove rows)
- Server-side registration, sanitization, and persistence of cws_core_field_groupings option
- inject_options() reads configured field names and builds dynamic cws_jobs_by_{field} groupings, replacing the hardcoded category/city logic
</objective>

In `inject_options()`, locate and REPLACE the entire hardcoded grouping block (from the `$by_category = array();` line through the closing `$this->plugin->log( sprintf( 'Grouped sets — %d categories, %d cities', ... ) )` call, inclusive) with the following dynamic implementation:

```php
// Dynamic field groupings — builds cws_jobs_by_{field} for each admin-configured field name.
// Usage: {#loop options.cws_jobs_by_primary_category.engineering as job}
$field_groupings = get_option( 'cws_core_field_groupings', array() );

if ( ! empty( $field_groupings ) && is_array( $field_groupings ) ) {
    foreach ( $field_groupings as $field_name ) {
        $field_name = sanitize_text_field( $field_name );
        if ( '' === $field_name ) {
            continue;
        }

        $grouped   = array();
        $option_key = 'cws_jobs_by_' . $field_name;

        foreach ( $options['cws_jobs'] as $job ) {
            $raw_value = isset( $job[ $field_name ] ) ? $job[ $field_name ] : '';

            // Only group by scalar string values — skip arrays/objects (e.g. location, permalink, raw_data).
            if ( ! is_string( $raw_value ) && ! is_numeric( $raw_value ) ) {
                continue;
            }

            $bucket_key = sanitize_title( (string) $raw_value );
            if ( '' !== $bucket_key ) {
                $grouped[ $bucket_key ][] = $job;
            }
        }

        $options[ $option_key ] = $grouped;

        $this->plugin->log(
            sprintf( 'Grouped set cws_jobs_by_%s — %d buckets', $field_name, count( $grouped ) ),
            'debug'
        );
    }
}
```

Key implementation notes:
- The Etch variable name is `cws_jobs_by_{field_name}` — e.g. field name "primary_category" → `{options.cws_jobs_by_primary_category}`
- `sanitize_title()` on the bucket key converts "Product Management" to "product-management" (same behavior as the old hardcoded logic)
- Skip non-scalar values (arrays/objects) — the `location`, `permalink`, and `raw_data` keys in format_job_data() are arrays and cannot be used as grouping sources; this guard prevents a PHP warning and silently skips unusable fields
- The old log message referenced "categories" and "cities" by name — the new per-field log is more informative and replaces it
- Do NOT add a fallback that auto-creates cws_jobs_by_category or cws_jobs_by_city — the admin must explicitly configure them to get those groupings
