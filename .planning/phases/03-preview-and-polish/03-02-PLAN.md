Anti-patterns to avoid:
- Do NOT use `date()` — it uses PHP server timezone, not WP site timezone
- Do NOT use `date_format( date_create( $raw ) )` — use `wp_date()` with `strtotime()`
- Do NOT skip the `?: ''` coercion — `wp_date()` returns `false` on bad input; Etch templates must receive strings not PHP false
- Do NOT use `empty()` check around `wp_date()` result — it would coerce '0' to empty string (unlikely for dates but incorrect). Use `?: ''` (null coalesce for false only)
- Do NOT modify the `'open_date'` or `'update_date'` raw keys — they must remain unchanged as existing templates may depend on them
