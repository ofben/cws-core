  <name>Task 3: Append stored query params to every API URL in build_api_url()</name>
  <files>includes/class-cws-core-api.php</files>
  <action>
In `build_api_url()`, after the existing `add_query_arg()` call that sets `organization` and `jobList`, add the following block before the `log` call and `return $url`:

```php
// Append admin-configured query parameters
$extra_params = get_option( 'cws_core_query_params', array() );
if ( ! empty( $extra_params ) && is_array( $extra_params ) ) {
    foreach ( $extra_params as $param ) {
        if ( ! empty( $param['key'] ) ) {
            $url = add_query_arg(
                sanitize_key( $param['key'] ),
                isset( $param['value'] ) ? urlencode( $param['value'] ) : '',
                $url
            );
        }
    }
}
```

Use `sanitize_key()` on the key (converts to lowercase alphanumeric + underscores + hyphens — appropriate for URL query param keys). Use `urlencode()` on the value so special characters are safe in the URL.

Note: `add_query_arg()` accepts a single key/value pair when called as `add_query_arg( $key, $value, $url )` — this is the correct form for iterating pairs one at a time.
