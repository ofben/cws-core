## Integration Gaps Found

### GAP-1: Fetch metadata not written on JSON parse failure / invalid response
**Severity:** Low — degrades display quality, does not break any flow
**File:** `includes/class-cws-core-api.php`, lines 174-178 (JSON parse) and 182-186 (invalid response)
**Description:** Both paths return `false` without calling `update_option` for `cws_core_last_fetch_time` or `cws_core_last_fetch_status`. When the API returns HTTP 200 with malformed JSON or unexpected structure, the admin cache status block retains stale values from the previous fetch.
**Affected requirements:** CACHE-01, CACHE-02, CACHE-04 (partial degradation for this edge case only)
**Recommendation:** Add `update_option()` calls at both failure paths, or add a dedicated failure code (e.g., status=-1 for JSON error, status=-2 for structure error).

### GAP-2: uninstall.php missing v1.1 options
**Severity:** Low — only visible after plugin deletion
**File:** `uninstall.php`, lines 26-31
**Description:** The `$options_to_delete` array omits all v1.1 options introduced in phases 5-8. Orphaned options remain in wp_options after plugin uninstall.
**Missing:** `cws_core_last_fetch_time`, `cws_core_last_fetch_status`, `cws_core_query_params`, `cws_core_field_groupings`, `cws_core_preview_job_id`
**Recommendation:** Add all five options to the delete array in uninstall.php.

## Breaking Change Notice

**Phase 7 removed hardcoded `cws_jobs_by_category` and `cws_jobs_by_city` Etch variables.**

Any existing Etch templates using `{options.cws_jobs_by_category}` or `{options.cws_jobs_by_city}` will receive empty/undefined values until the admin adds the corresponding field names to the Field Groupings settings page.

**Migration required:**
1. Admin visits Settings > CWS Core > API Configuration > Field Groupings
2. Add `primary_category` and/or `primary_city` (or the exact field name from the API response) as grouping sources
3. Save settings
4. Etch templates using `{options.cws_jobs_by_primary_category}` etc. will now work

This is intentional — the dynamic system replaces the hardcoded approach to allow any field to be used as a grouping, not just category and city.
