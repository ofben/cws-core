Purpose: Without this, editors see blank `{options.cws_job.*}` fields while designing the job template — they cannot visually verify their template against real data.

Output: `includes/class-cws-core-etch.php` with a new `elseif` preview branch at the top of `handle_single_job()`. The branch detects the builder context, fetches the first configured job via the existing API, and populates `$this->current_job` — which `inject_options()` already reads and injects, so no changes to `inject_options()` are needed.
